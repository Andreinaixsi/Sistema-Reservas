# Requerimientos No Funcionales

| ID | RF |Descripción | Criterio |
| :--- | :--- | :--- | :--- | :--- |
| **RF-001** | **Usabilidad** |  El sistema debe ser fácil de usar, con una interfaz clara e intuitiva que permita a los usuarios realizar acciones sin necesidad de capacitación previa.| 1. Navegación simple con máximo 3 clics para crear una reserva. 2.Formularios claros con validación de errores visibles. 3.Diseño responsivo para uso en computadoras y dispositivos móviles.  Mensajes de error y confirmación comprensibles.|
|**RF-002** | **Rendomiento** | El sistema debe responder de manera rápida a las solicitudes del usuario para garantizar una buena experiencia.| 1.Tiempo de respuesta menor a 2 segundos en operaciones comunes (login, listar recursos, ver reservas). 2.Creación de reservas en menos de 3 segundos. 3.Soporte mínimo de múltiples usuarios simultáneos sin degradación notable. |
| **RF-003** | **Seguridad** |El sistema debe proteger los datos de los usuarios y garantizar el acceso controlado a la información. | 1.Autenticación mediante JWT. 2.Contraseñas almacenadas con hash (bcrypt). 3.Un usuario solo puede acceder a las organizaciones a las que pertenece. 4.Validación de permisos para acciones críticas (crear recursos, cancelar reservas). 5.Protección contra acceso no autorizado a datos de otras organizaciones.|
| **RF-004** | **Disponibilidad** | El sistema debe estar disponible para su uso en la mayoría del tiempo durante el periodo de evaluación. | Disponibilidad mínima del 95% durante pruebas y demostración. Capacidad de reinicio rápido en caso de fallo. Uso en entorno local o servidor accesible para demo. |
| **RF-005** | **Mantenibilidad** | El sistema debe estar estructurado de forma que facilite futuras modificaciones y ampliaciones. | Uso de arquitectura modular (Screaming Architecture). Separación clara entre frontend, backend y base de datos. Código organizado por funcionalidades (users, reservations, resources). Documentación actualizada del sistema. Uso de nombres claros en variables, funciones y archivos. |
